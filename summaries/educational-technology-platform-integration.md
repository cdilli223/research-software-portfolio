# Educational Technology Platform Integration and Assessment Infrastructure

## Summary

Employer-owned work involving Canvas, Learnosity, platform migration, assessment workflows, QA, documentation, and cross-functional implementation support.

This work focused on replacing legacy instructional and assessment workflows with more scalable platform infrastructure, supporting curriculum delivery, assessment operations, integration planning, user adoption, and documentation.

## Context

The work sat at the intersection of product ownership, platform implementation, educational technology, assessment systems, and operational support. The goal was to help move instructional and assessment workflows from legacy tools toward systems that could better support classroom delivery, assessment authoring, student response workflows, and reporting.

Because the work was employer-owned, this summary does not include private implementation details, internal data, credentials, or proprietary workflows.

## Technical Problem

The platform work needed to translate instructional and assessment requirements into reliable technical workflows across multiple systems. That meant validating that tools supported the intended classroom and assessment use cases, documenting operational processes, and identifying issues before they reached school-based users.

The challenge was not only implementation, but coordination: curriculum, product, operations, school teams, and vendors needed shared expectations about what the platform should do and how success would be verified.

## My Contributions

I worked on:

- Canvas implementation and workflow design
- Learnosity assessment workflow support
- platform migration planning
- assessment readiness and QA
- documentation for users and internal teams
- cross-functional coordination with curriculum, product, operations, and school-based teams
- identifying integration and adoption risks
- translating instructional needs into technical workflow requirements

## Validation and Evidence

Validation involved:

- assessment-readiness checks
- workflow testing
- user-facing documentation review
- cross-functional review with content and implementation teams
- issue triage during rollout
- verification of assessment and platform behavior against expected instructional workflows

## What I Would Improve

If revisiting the work with more engineering ownership, I would formalize reusable QA checklists, create clearer acceptance criteria for each workflow, document integration assumptions more explicitly, and build lightweight monitoring or reporting around recurring support issues.
